# Starknet Full Node Configuration

Configuring your Starknet full node properly ensures that it runs efficiently and connects correctly to the Starknet network. Configuration can be done through three main avenues:

- Command-line arguments
- Environment variables
- Configuration files

## Priority of Configuration

<Callout type="info" emoji="â„¹ï¸">
  Configuration priority is as follows: command-line arguments > environment variables > configuration files. When the same setting is configured in multiple places, the source with the highest priority takes effect.
</Callout>

## Command-Line Options

For a comprehensive list of command-line options:

```bash
starknet-full-node --help
```

Below are some essential command-line options and a categorized list of advanced configurations:

### Basic Command-Line Options

- **`-d, --base-path <path>`**: Set the directory for Starknet data (e.g., databases, configurations).
  
- **`-c, --config <file>`**: Specify a configuration file.

- **`--chain <chain>`**: Select the blockchain configuration (e.g., mainnet, testnet).

- **`--execution-strategy <strategy>`**: Choose the execution strategy for transactions.

- **`--pruning <method>`**: Define the database pruning strategy to manage disk space.

- **`--validator`**: Run the node in validator mode, if applicable.

- **`-v, --version`**: Display the software version.

## Environment Variables

Set up your node's environment variables using the `STARKNET_` prefix. For example:

- `STARKNET_BASE_PATH=/path/to/data`
- `STARKNET_LOG=info`

These variables allow you to adjust the node's configuration without using command-line arguments.

## Configuration File

You can use a JSON, TOML, or YAML file to structure your configuration settings. Specify your configuration file on startup with the `-c` option. Here's a basic example in JSON format:

```json
{
  "base_path": "/path/to/data",
  "chain": "mainnet",
  "network": {
    "port": 30303,
    "bootnodes": ["node1.example.com", "node2.example.com"]
  },
  "database": {
    "db_cache": 1024,
    "state_cache_size": 128
  },
  "log": {
    "level": "info",
    "filters": "starknet=trace"
  }
}
```
